## How to Contribute

### 1. Report Issues

Found a bug or have a suggestion? Open an issue on GitHub.

**Please include:**
- Clear description of the issue or suggestion
- Steps to reproduce (if applicable)
- Expected vs actual behavior
- Screenshots (if UI-related)

### 2. Submit Pull Requests

**Process:**
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Test thoroughly
5. Submit a pull request with a clear description

### 3. Contribute to Lore

The QZEN universe is built on stories. Contribute to the lore:
- Write Council narratives
- Create artwork or visual concepts
- Propose new lore chapters
- Expand the Manuscripts of Zenal

### 4. Community Building

- Help moderate Discord and Telegram
- Create educational content
- Translate documentation
- Onboard new citizens

## Code Standards

### Smart Contracts
- Solidity ^0.8.20
- OpenZeppelin base contracts
- 100% test coverage required
- Security-first approach
- No admin functions or backdoors

### Website
- Next.js + TypeScript
- Tailwind CSS for styling
- Framer Motion for animations
- Accessible (WCAG 2.1 AA)
- Mobile-first responsive design

### General
- Clean, readable code
- Meaningful commit messages
- Documentation for all public APIs
- No hardcoded values

---

## Commit Message Format

```
type: brief description

Detailed explanation if needed.
```

**Types:**
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation update
- `style:` Formatting, no code change
- `refactor:` Code restructuring
- `test:` Adding or updating tests
- `chore:` Maintenance tasks

## Getting Started

### Prerequisites
- Node.js 18+
- npm or yarn
- Git

### Local Development
```bash
# Clone the repository
git clone https://github.com/QZENOfficial/QZEN.git
cd QZEN

# Install dependencies (website)
cd website
npm install
npm run dev
```
